Databases Roadmap

Learn how apps store, read, and sync data — from your phone to the cloud. Woven into Month 5 (Days 135–150) and used in Month 6.

DB Track

🧠 What Types of Databases Will You Use?

1. On-Device (Offline) Storage — data saved directly on the phone. Fast. Works offline.
   → AsyncStorage, MMKV, SQLite, Realm

2. Cloud Database (Online) — data saved on a server. Syncs between devices.
   → Firebase Firestore, Supabase (PostgreSQL)

3. Relational (SQL) — structured tables. Best for complex data with relationships.
   → PostgreSQL (Supabase), SQLite

4. NoSQL (Document) — flexible JSON-like documents. Easy to start.
   → Firebase Firestore

  • Row / Document — one record (e.g. one movie)

-- users table CREATE TABLE users ( id INTEGER PRIMARY KEY, name TEXT, email TEXT ); -- favorites table (links users to movies) CREATE TABLE favorites ( id INTEGER PRIMARY KEY, user_id INTEGER, movie_id INTEGER ); -- Get all favorite movies for user with id = 1 SELECT users.name, movies.title, movies.rating FROM favorites JOIN users ON favorites.user_id = users.id JOIN movies ON favorites.movie_id = movies.id WHERE users.id = 1;

⚖️ Firebase vs Supabase — Which to Choose?

Firebase Firestore — NoSQL, easier to start, Google ecosystem, great for real-time apps. Choose for: chat features, real-time updates.

Supabase — SQL (PostgreSQL), open-source, more powerful queries, built-in Auth. Choose for: structured data, complex filtering, row-level security.

For your Movie App: Either works great. Supabase is recommended because it teaches you real SQL which employers value highly.

💡 Key Concepts

  • SQL (Structured) — data in rows and columns like Excel. Used in: Supabase, MySQL, PostgreSQL
  • NoSQL (Flexible) — data in documents like JSON objects. Used in: Firebase, MongoDB
  • CRUD — Create, Read, Update, Delete. Every database does these 4 things
  • Primary Key — a unique ID for every record (like movie.id)
